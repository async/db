From prototype to production

A database made of JSON files.

Start with JSON files in db/. @async/db infers schema metadata, writes TypeScript types, and serves local REST, GraphQL, and a local data explorer while your app contract takes shape.

This docs site uses @async/db content collections to load markdown from docs/ through the allowlist in website/db.schema.js.

Getting started Production guides

install

Start from a package install.

pnpm add @async/db
pnpm exec async-db init
pnpm exec async-db serve

Keep serve running, then open the local data explorer at http://127.0.0.1:7331/__db.

npm install @async/db
npx async-db init
npx async-db serve

Use npx to run the local package binary after install.

Keep local routes local, then expose reviewed operations.

Use JSON-shaped routes while the data shape is moving. When the app-facing contract settles, move browser traffic to registered operation refs, app-owned auth, and explicit route exposure rules.

Step Guide
registered operations Prototype to production
server.expose Server settings
contracts + doctor Schema strictness
store graduation Mixed stores
GET /db/users.json POST /__db/operations/GetUserProfile server.expose.rest = "registered-only" server.expose.viewer = "dev"

Operation refs are allowlist identifiers, not secrets. They keep route shape reviewable while normal app policy still owns authentication, authorization, limits, and monitoring.
