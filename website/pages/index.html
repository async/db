JSON files now, contracts when they matter

A database made of JSON files.

Start with records in db/. @async/db infers useful local contracts, writes generated TypeScript types, and serves REST, GraphQL, and a local data explorer while your app shape is still moving.

The workflow is data-first by default. Add explicit schema only when the contract needs descriptions, defaults, enums, relations, or stricter validation.

Getting started Production guides Package API 
pnpm add @async/db
pnpm exec async-db init
pnpm exec async-db serve
1. Seed data Write JSON, JSONC, or CSV files under db/.
2. Sync contracts Generate schema metadata, TypeScript types, and runtime state under .db/.
3. Serve locally Use REST, GraphQL, and the local data explorer while the app contract settles.

One File Is Enough To Start

A plain data file teaches @async/db the first useful shape. No schema is required for the first pass.

db/users.json

[
  {
    "id": "u_1",
    "email": "ada@example.com"
  }
]

generated type

export type User = {
  id: string;
  email: string;
};

What Sync Writes

Output Purpose
.db/schema.generated.json Normalized schema metadata inferred from data files and explicit schema sources.
.db/types/index.d.ts Generated TypeScript contracts for local app code and editor feedback.
.db/state/*.json Writable runtime state refreshed from source hashes without rewriting source files.

Local First, Then App Backed

Keep JSON-shaped routes open while prototyping. When the app-facing contract settles, graduate browser traffic to registered operation refs, app-owned auth, and explicit route exposure rules.

Prototype

  • GET /db/users.json
  • POST /db/users
  • http://127.0.0.1:7331/__db

Production path

  • Registered operations
  • Route exposure controls
  • SQLite, Postgres, or app APIs per resource
